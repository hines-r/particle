Pixel Chaos

  • C#
  • Unity®

Pixel Chaos was made in the Unity game engine and is by far my favorite personal project because it was one of the most fun to make. It's a defend the castle style game featuring endless waves of enemies and real-world physics calculations. To play, the player must purchase units to place on a 3x3 grid that can attack enemies. There are 13 unique units with two distinct upgrade paths each. Pixel Chaos made me realize just how powerful mathematics and programming can be. It simulates gravitational force, and I needed to calculate several different projectile motion problems to make it all possible. For example, calculating the force required to fire a projectile at a moving target. Other problems included programming bounce on a simulated 3D plane and utilizing scaling algorithms to adjust and balance the infinite waves.

Virtual Woodcutter

  • Java
  • Android
  • libGDX

I've always thought incremental games were an interesting concept. There's typically no significant gameplay element besides the player performing a simple repetitive task. Doing so results in the player earning some form of currency and in the case of Virtual Woodcutter, it's wood and gold! To gather wood, the player can tap the screen to chop down procedurally generated virtual trees. Occasionally, a bird carrying a power-up will traverse the screen and if tapped, grants the player a wide array of powers including extra strength, auto chopping, and increased log gain. To earn gold, the player can expend logs at workshops. Gold can be used to purchase upgrades, more workshops, and workers that collect logs passively. I learned a lot from this project including new and useful design patterns like game states and object pooling. It taught me why its imperative to optimize code, especially on mobile because of the slower hardware. I put a lot of love into this project, and it was a pleasure to make!

AniNeko

  • C#
  • SQLite

AniNeko is an application meant to index and keep track of anime, TV series, or movies. It's a WPF application featuring an attractive material design built using the Material Design in XAML Toolkit. It uses an embedded SQLite database to store records locally and allows users to add, edit, delete or search for entries as needed. Users also have access to tabs at the top of the screen to filter entries by their current watch status. Architecturally, it's built using an MVVM (Model-View-ViewModel) pattern to separate the logic from the GUI. I was inspired to make this program to solve a problem I once had. That being keeping track of the many shows I would end up watching. With this program, logging progress becomes easy and its a more purpose-built solution than writing it out in a text document like I once did.

Home Database Tool

  • Java
  • SQL

This program was a group project I completed while in university. Its purpose is to search, store and categorize homes or even locations by their current condition or state of being. It works by utilizing an SQL database to store entries by pressing the buttons on the right side of the GUI. The maps are generated using the Google Maps API, and the user can toggle between street view or map view via the right side toggle buttons. Depending on the view mode, there are also two sliders the user can control. When in street view mode, there is a field of view adjustment slider and an angle slider. They allow the user to adjust the 360° camera at any given position. While in map mode, the user has access to a zoom slider to zoom in or out of the image. I was in charge of implementing the GUI and getting the API integrated into the program. It was a great project, and I learned a lot from it.
